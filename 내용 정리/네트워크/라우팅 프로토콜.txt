라우팅 프로토콜

정적 : 관리자가 직접 부여, 부하가 적음, 네트워크변화에 민감하게 대응 불가
	- static : 단일 목적지에 방향을 넣음
	- default : 지정하지않은 나머지에 방향을 넣음

동적 : 셋팅 후 자동, 부하가 많음, 네트워크변화에 민감하게 대응 가능 
	- AS : 자치시스템(관리자에 의해 관리되는 라우터의 집합)
	- ASBR : 다른 AS끼리 연결할때 사용되는 라우터

	- IGP
	   - distance vertor : RIPv2
	     ○ 전체 네트워크를 학습하여 최적의 경로를 산출(Hop 단위를 이용)

	   - link-state : OSPF
	     ○ 대역폭(연결상태정보를 확인)을 사용하여 최적의 경로를 산출 
	      	
	- EGP 
	   - BGP
